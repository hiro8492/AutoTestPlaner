次の対応候補（closed版関連）

1. ポータブルexe（インストーラ不要）対応
   - 目的: USB運用や権限制限環境での利用を容易にする。
   - 作業概要: electron-builder の target を portable に追加し、配布物を dist に出力。
→後回し

2. アイコン／アプリ名／バージョン整備
   - 目的: 配布・社内展開時の見栄えと識別性を向上。
   - 作業概要: icon設定、productName、description/author を package.json に追加。

3. LLMインストーラ連携（初回起動チェックや案内）
   - 目的: Ollama未導入時のエラーを減らし、導入手順を明確化。
   - 作業概要: 起動時に http://localhost:11434 を確認し、未起動なら案内ダイアログ。

4. 自動更新／署名の検討
   - 目的: 配布後の更新を安全に行い、Windowsの警告を軽減。
   - 作業概要: 署名証明書の導入、auto-updater 構成の検討。

5. 初回起動時のDB初期化フロー改善
   - 目的: ユーザーごとの初期データ作成やサンプル投入を明確化。
   - 作業概要: 初回起動時に初期プロファイルやサンプルIRを作成する処理を追加。

推奨事項（進め方）
- 優先度1: LLMインストーラ連携（3）。実運用でのトラブルを最も減らせる。
- 優先度2: アイコン／アプリ名／バージョン整備（2）。配布時の品質を上げる。
- 優先度3: ポータブルexe（1）。導入環境が多様な場合に効果的。
- 署名（4）は配布対象が社外の場合に特に推奨。

備考
- 現在の closed版 は NSIS インストーラで作成済み。
- UI/Backend はローカルで起動し、APIは 127.0.0.1:3001 を使用。
